^Slanted edge

Perhaps viewing this image will give you a hint on how to have a slanted-edge graphical bar:

Here, the numbers to the left of each tile indicates an index of the tile array, while the numbers on the right are the fill amount. Combine this image with the one below:

and you get the idea on how it works. Technically, the amount of fill to be converted into tile index is exceeding the number of pieces to accommodate the next tile's fill amount. Also notice that since there is no next tile after the right end, the number of pieces can safely be set to the number of distinct fill amounts.

Duplicate tile numbers on the last n tiles are needed since this covers all possible values of the next tile. If there are 2+ possible different tiles (such as after the left end is 8 pieces for each middle tile or if the bar has no middle tiles and a 6-piece right end tile), you would take the number of whichever has the highest value (in this case, you would take 3 pieces for left end + 8 for middle which means the highest index would be $0B (11) instead of 3 + 6 = $09 (9) so that both the left end and middle would not use glitched tiles)
